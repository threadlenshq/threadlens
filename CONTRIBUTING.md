# Contributing to ThreadLens Open-Core

Public contributions are accepted for the `open-core/` workspace through GitHub forks and pull requests only.

## Before you open a pull request

- Review the license in [`LICENSE`](LICENSE).
- Read the public docs contribution guides in [`docs/src/content/docs/contributing/`](docs/src/content/docs/contributing/).
- Keep changes scoped to `open-core/` unless a maintainer explicitly asks for a wider workspace change.

## Contribution flow

1. Fork the repository.
2. Create a branch in your fork.
3. Make your change.
4. Run the relevant checks from `open-core/`.
5. Open a pull request for review.

## Contribution terms

By submitting a pull request for material intended to be merged into `open-core/`, you represent that:

- You have the legal right to submit the contribution.
- The contribution is your original work, or you have the right to pass it on under these terms.
- You are submitting the contribution for ThreadLens open-core to use, reproduce, modify, distribute, sublicense, and relicense as part of the project.
- You understand that `open-core/` is source-available under BUSL 1.1 today and may convert to the Change License described in [`LICENSE`](LICENSE).

If you do not agree with these terms, do not submit the contribution.

## High-value contributions

- Reproducible bug fixes.
- Docs improvements.
- Self-hosting setup improvements.
- Pipeline correctness fixes.
- Small product-quality improvements that keep the open-core app simple to run.
